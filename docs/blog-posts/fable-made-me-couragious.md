For months, I held back from building Level 3 into my intelligent textbook platform.

Level 3 — "adaptive textbooks" — is the stage where a book stops being static content and starts learning about the learner: capturing every MicroSim interaction as an xAPI statement, storing it in a Learning Record Store, and using that evidence to predict which concepts a student has actually mastered.

It's also the stage where the stakes change. You're no longer just publishing pages — you're storing data about individual students. If that data isn't handled with real security discipline, it's teachers and schools who take the risk, not me.

That's what kept me at "Level 2.99" — generating fun, genuinely useful AI-authored textbooks, but stopping short of the adaptive layer that makes them truly intelligent.

What changed: Claude Fable 5. Having a model I trust to sit beside me and run a rigorous security review of my Learning Record Store — the ingestion pipeline, the event store, the access controls — gave me the confidence to build the thing I'd been avoiding.

So I'm moving forward: capturing MicroSim events, modeling concept mastery with Bayesian Knowledge Tracing, and turning a library of static chapters into something that actually adapts to each learner — without cutting corners on the privacy and security Level 3 demands.



Thank you, Fable, for the courage to make the leap.
